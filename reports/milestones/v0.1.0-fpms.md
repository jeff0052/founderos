## 背景

FounderOS 需要一个项目管理引擎，能支撑一个人+AI团队管理多条业务线。市面上的工具（Jira、Linear）都是为人类团队设计的，不适合 AI Agent 直接操作。

## 目标

从零构建一个 AI-native 的项目管理系统：
- Agent 通过 tool call 直接操作，不需要 UI
- 分形结构（项目→任务→子任务无限嵌套）
- DAG 依赖关系 + 状态机
- 轻量级，SQLite 单文件

## 交付物

- **16 个模块：** store, tools, validator, dashboard, focus, bundle, compression, heartbeat, models, command_executor, archive, narrative, rollup, recovery, risk, schema
- **3034 行代码**
- **494 个测试，全绿，3.39s**
- **spine.py CLI** 接入 OpenClaw
- **Dashboard** GitHub Pages 可视化

## 关键决策

1. **SQLite + WAL** 作为 source of truth — 简单可靠，单机够用
2. **所有写入通过 Tool Call** — LLM 不直接碰存储层
3. **眼球模型 L0/L1/L2** — 三级分辨率控制 context 大小
4. **原子提交** — 每个操作要么完全成功要么完全回滚

## 踩过的坑
