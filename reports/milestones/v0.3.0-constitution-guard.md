# v0.3.0 — Constitution Guard 三道物理防线

**日期：** 2026-03-18
**状态：** ✅ 完成

---

## 背景

CTO Agent 有了 22 条铁律（CONSTITUTION.md），但铁律只是文字。AI Agent 可能无意中违反——比如在支付路径中用 float 而不是 Decimal，或者悄悄修改关键测试。需要把铁律变成代码级别的硬约束。

## 目标

将 Constitution 中最关键的 3 条规则变成自动化检查，从"靠自觉"升级为"靠机器"。

## 交付物

- **AST Float Scanner** — 检测支付路径中的 float 使用（Constitution §13）
- **Ironclad Test Lock** — SHA-256 哈希锁定铁律测试，防篡改（Constitution §5）
- **Core Path Gate** — 标记支付核心路径变更，需 Founder 审批（Constitution §3）
- **constitution_guard CLI** — 统一入口（check / update-hashes / status）
- **CODE-REVIEW-STANDARD.md** — P0-P3 分级审查标准
- **GIT-WORKFLOW.md** — 分支策略 + commit 规范
- **34 个新测试** → 总计 533 tests

## 关键决策

1. **AST 分析而非正则** — 准确率高，不会误报字符串中的 "float"
2. **SHA-256 而非文件锁** — 哈希比权限更可靠，且可审计
3. **Gate 只标记不阻止** — 保持开发流畅，但确保 Founder 知情

## 踩过的坑

- S1: recovery.py bundle.assemble() 位置参数错误 → 改为关键字参数
- S2: validator.py 每次校验新开 DB 连接 → 复用 store._conn
- S3: store.py get_connection() 不建表 → 改为 init_db()
