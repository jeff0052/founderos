# FPMS — 分形项目管理系统

你不需要学任何命令。直接跟我说话就行。

---

## 怎么用

### 随口扔待办

> "记一下，明天打电话给会计"  
> "帮我记住 NetStar 合同要续期"

我会自动创建节点放进 inbox。

### 建项目

> "新建一个项目：Anext 信贷，目标是拿到 700 万额度"  
> "Anext 下面加两个任务：法务审核、利率谈判"

我会创建层级结构 goal → task，自动挂载。

### 问进度

> "Anext 卡在哪了？"  
> "现在手上有什么事？"  
> "看板"

我会调出全局看板，告诉你每个项目的状态和风险。

### 推进状态

> "法务审核完了"  
> "利率谈判开始做了"  
> "Anext 信贷先搁一下"

我会更新状态（done / active / dropped），自动检查前置条件。

### 加依赖

> "利率谈判要等法务审核完才能开始"

我会建立依赖关系，自动标记 blocked。

### 记进展

> "法务那边说担保条款有风险，要改"

我会把这条追加到对应节点的叙事里。

### 设 deadline

> "法务审核下周三前要搞定"

## 你会看到什么

### 看板（自动推送）

```
📥 明天打电话给会计
📥 NetStar 合同续期

📁 Anext 信贷 [🔵active] 🚨at-risk
  ├─ 法务审核 [⏳waiting] 🚫blocked
  ├─ 利率谈判 [🔵active] 🚨deadline:3/18
  └─ 内部审批 [✅done]
```

### 告警（心跳自动扫描）

```
🔴 [task-xxxx] urgent_deadline: 法务审核 deadline < 24h
🟡 [task-yyyy] stale_warning: NetStar 技术对接 7 天没动
```

### 风险标记

| 标记 | 含义 |
|------|------|
| 🚫 blocked | 有事情卡着，等前置完成 |
| 🚨 at-risk | deadline 不到 48 小时 |
| 💤 stale | 超过 7 天没推进 |

## 节点类型

| 类型 | 用途 | 例子 |
|------|------|------|
| goal | 战略目标 | "Q2 完成 A 轮融资" |
| project | 项目 | "Anext 信贷" |
| milestone | 里程碑 | "完成法律文件" |
| task | 具体任务 | "审核担保条款" |
| unknown | 先记下，以后整理 | "那个什么事要处理一下" |

---

## 状态流转

```
inbox（收件箱）
  ↓ 整理后
active（进行中）→ done（完成）
  ↓               ↑ 需要理由
waiting（等待中） → dropped（放弃）→ inbox（恢复，需要理由）
```

- **inbox**: 随手记的，还没整理
- **active**: 正在推进
- **waiting**: 等别的事完成
- **done**: 完成了
- **dropped**: 不做了

---

## 背后原理（不需要懂，但可以知道）

- 所有数据存在本地 SQLite + Markdown 文件，不上云
- 风险标记是实时计算的，不是手动设的
- 每次操作都有审计记录（events.jsonl）
- 系统自动归档完成超 7 天的节点，释放注意力空间
- 心跳定期扫描，主动提醒你有什么卡住了
