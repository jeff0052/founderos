# FounderOS 白皮书 V2

*A Control System for Founders*

---

## 一、愿景（Vision）

FounderOS 是一个帮助创始人理解、控制和进化公司的系统。

在 AI 时代，计算能力会逐渐替代大部分执行工作，而 Founder 的核心价值将集中在三个方面：
- Vision（方向）
- Judgment（判断）
- Trust（被信任的个体）

FounderOS 的使命是：

**放大 Founder 的认知能力和决策能力。**

FounderOS 不是传统的软件工具，而是一个 **控制系统（Control System）**。

---

## 二、核心思想

FounderOS 将公司视为一个 **动态系统（Dynamic System）**。

这个系统不断与环境互动，并通过反馈循环不断调整。

核心公式：

**State + Signal → Decision → Action → New State**

系统循环：

```
Environment
 ↓
Signals
 ↓
Founder
 ↓
Decisions
 ↓
Missions
 ↓
State Update
```

这个循环不断运行，使公司像一个 **生物系统一样演化**。

---

## 三、FounderOS 四个核心模块

1. **State**（状态）
2. **Signals**（信号）
3. **Decisions**（决策）
4. **Missions**（任务）

---

## 四、State（系统状态）

State 是公司当前的客观状态。

所有 State 必须满足两个条件：
- 可量化
- 可长期跟踪

每个系统只保留 2–3 个核心指标。

### 示例

| 系统 | 指标 |
|------|------|
| Uniweb | Merchant Count, Monthly GMV, Net Profit |
| Onta | Active Nodes, Daily Transactions, SDK Integrations |
| License | License Count, Application Status, Compliance Progress |

---

## 五、Signals（外部信号）

Signals 是来自外部世界的重要变化。

来源：市场、合作伙伴、投资人、技术、政策监管、用户反馈

特点：不确定、不连续、可能改变战略

### 示例

- Binance wallet partnership discussion
- Philippines partner ready for live testing
- Stablecoin regulation update
- Investor strategic suggestion

Signals 的作用：**帮助 Founder 感知环境变化。**

---

## 六、Decisions（Founder决策）

Founder 根据当前 State + 外部 Signals 做出关键决策。

**Founder 每周只做少量决策（≤3）。**

### 示例

- Priority 1: Philippines real transaction test
- Priority 2: Start Sunmi SDK integration
- Priority 3: Prepare Binance wallet integration

Decisions 的作用：**将复杂环境压缩为清晰方向。**

---

## 七、Missions（执行任务）

Mission 是决策的执行层。每个 Mission 包含：Mission / Owner / Deadline / Result

### 示例

| Mission | Owner | Deadline | Result |
|---------|-------|----------|--------|
| Philippines real transaction test | Ted | March 15 | Complete first real payment |

Mission 的目标：**把决策转化为可执行行动。**

---

## 八、FounderOS 更新流程

每周更新一次，30–60分钟。

更新顺序：
1. Update State
2. Record Signals
3. Make Decisions
4. Update Missions

---

## 九、FounderOS Civilization Map

长期层面定义 **文明等级（Civilization Levels）**。

当关键指标达到某个阈值，系统进入新的阶段。

### 示例（支付文明）

| Level | 阶段 |
|-------|------|
| Level 1 | Offline acquiring |
| Level 2 | Regional payment network |
| Level 3 | Stablecoin settlement network |
| Level 4 | Global payment infrastructure |

文明层的作用：战略叙事、团队方向、投资人沟通

---

## 十、FounderOS UI 结构

界面结构类似一个 **驾驶舱（Cockpit）**：

1. **State Dashboard** — 核心指标一览
2. **Signals Panel** — 外部信号流
3. **Decision Panel** — 本周决策（≤3）
4. **Mission Board** — 任务看板

---

## 十一、三个原则

1. **极简** — 只关注状态、信号、决策、任务
2. **手动更新** — V1 采用手动，强化感知，避免过载
3. **Founder Control System** — 本质不是管理工具，是控制系统

---

## 十二、终极形态

| 版本 | 形态 |
|------|------|
| V1 | 手动系统 |
| V2 | AI 辅助分析 |
| V3 | Agent 执行系统 |
| V4 | 自动化公司操作系统 |

---

## 十三、最终目标

让一个 Founder 可以管理一个越来越复杂的系统。

- 人类提供 Vision
- AI 提供 Execution
- FounderOS 提供 Control
