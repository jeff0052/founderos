# 7. 开放问题

## 7.1 已在本文档中解决

| 原始问题 | 解决方案 | 出处 |
|----------|----------|------|
| 节点 ID 规则 | 语义前缀 + 短哈希（如 `task-7f2a`），系统生成 | FR-1 |
| 归档策略 | 终态 + 可配置冷却期（默认 7 天）+ 无未归档子节点/依赖方 + is_persistent 豁免 | FR-6 |
| 跨项目依赖 | depends_on 弱边，跨项目无限制，DAG 环路检测保护 | FR-5 |
| 优先级 | 无显式优先级字段；通过 deadline + status + 风险标记 + depended_by 爆炸半径隐含推导；焦点调度器按规则仲裁 | FR-5, FR-9 |
| 叙事体深度 | 按 event_type 枚举控制：status_change / decision / blocker / correction 必记，info 可选。记录业务事件而非每次交互 | FR-2 |
