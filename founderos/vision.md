# FounderOS Vision

_人类经营契约，AI快速演化实现，自动化验证守门。_

---

## 一、FounderOS 是什么

FounderOS 是一个帮助创始人理解、控制和进化公司的系统。

它不是传统的软件工具，不是项目管理系统，不是 CRM，不是 ERP。

**FounderOS 是一个控制系统（Control System）。**

在 AI 时代，计算能力会逐渐替代大部分执行工作，而 Founder 的核心价值将集中在三个方面：

- **Vision**（方向）— 看见别人看不见的未来
- **Judgment**（判断）— 在不确定性中做出正确选择
- **Trust**（信任）— 作为被信任的个体连接资源

FounderOS 的使命是：**放大 Founder 的认知能力和决策能力。**

---

## 二、核心思想

FounderOS 将公司视为一个**动态系统（Dynamic System）**。

这个系统不断与环境互动，并通过反馈循环不断调整。这与控制论（Cybernetics）的核心理论一致——生物、公司、社会，它们的共同结构都是：输入 → 控制 → 输出 → 反馈。

### 核心公式

```
State + Signal → Decision → Action → New State
```

### 系统循环

```
Environment（外部世界）
     ↓
  Signals（信号感知）
     ↓
  Founder（判断决策）
     ↓
  Decisions（战略决策）
     ↓
  Missions（任务执行）
     ↓
  State Update（状态更新）
     ↓
  Environment...（循环）
```

这个循环不断运行，使公司像一个**生物系统**一样演化。

## 三、双层世界模型

FounderOS 的核心架构是一个双层模型：

### 外部：世界地图（World Map）

AI 构建并维护一个完整的外部世界地图，包括：
- **支付网络全景** — 全球支付基础设施的当前状态
- **各国监管环境** — 牌照要求、政策变化、合规门槛
- **竞争对手态势** — 谁在做什么、谁在崛起、谁在衰落
- **技术门槛变化** — 新技术、新协议、新标准
- **合作伙伴网络** — 潜在合作方、已有关系、机会窗口

世界地图是 Signal 系统的可视化。它帮助 Founder 感知环境变化，做出战略判断。

### 内部：公司即身体（Company as Body）

公司被视为一个生物体：
- **Jeff = 大脑** — 感知信号、做出决策、设定方向
- **团队 = 身体** — 执行任务、交付结果、反馈数据
- **AI = 杠杆放大器** — 同时放大大脑的认知能力和身体的执行能力

内部系统管理：交易量、技术开发、团队状态、资金状况。各种信号回来之后，让大脑去做决策。

## 四、FounderOS 本质是一个"战略游戏"

Founder 就像一个 4X 策略游戏（Civilization、Stellaris、Crusader Kings）的玩家。

你每天在做的事情本质上是：
- **派任务**（Deploy Missions）
- **升级能力**（Upgrade Capabilities）
- **占领地图**（Expand Territory）
- **建立结构**（Build Governance）
- **控制资源**（Manage Resources）

但你的游戏地图不是世界，而是：**公司文明。**

### 文明地图

```
支付文明
  │
  ├── Uniweb（现金流文明）
  │   ├ 商户增长
  │   ├ 收单稳定
  │   └ 成本控制
  │
  ├── License（合规护城河）
  │   ├ 新加坡 DPT
```

### 文明等级（Civilization Levels）

当关键指标达到某个阈值，系统进入新的阶段：

| Level | 阶段 | 标志 |

| Level 1 | Offline acquiring | 线下收单，建立现金流 |

| Level 3 | Stablecoin settlement network | 稳定币结算网络，协议标准 |

## 五、三层文明架构

Jeff 定义了公司战略的三层结构：

- **Layer 1 生存层：Uniweb**（真实牌照 / 商户 / 现金流）
- **Layer 2 控制层：Founder OS / Agent 公司**（任务 / 决策 / 记忆 / 预警）
- **Layer 3 文明层：Onta**（支付网络协议 / 全球扩张）

**核心洞察：三者不是三家独立公司，而是一个三段式文明结构。**

- Uniweb = 现实现金入口
- Founder OS = 组织控制系统
- Onta = 未来支付网络协议

## 六、AI 在 FounderOS 里的角色

很多人做 FounderOS 会犯一个错误：**以为 AI 是核心。**

其实不是。AI 只是：**执行代理。**

```
Founder（决策）
  ↓
治理结构（方向）
  ↓
AI Agent（执行）
  ↓
结果（反馈）
```

AI 帮你：查资料、写代码、分析数据、自动执行、监控世界。

但治理权在 Founder。

### AI 杠杆的三个维度

**认知杠杆：** Jeff的直觉 × AI的算力 = 超人级判断力
**执行杠杆：** Jeff的一个想法 × AI军团执行 = 瞬间全球部署
**网络杠杆：** AI管理的合作伙伴网络 × 协议标准 = 无摩擦扩张

## 七、FounderOS 的三个原则

### 1. 极简
只关注：状态、信号、决策、任务。不追求复杂功能。

### 2. Founder Control
FounderOS 的本质不是管理工具，而是 Founder 的公司控制系统。治理权永远在人类手中。

### 3. 必须极简到每天都会打开
如果 FounderOS 每天打开一次，它就活着。如果三天不看，它就死了。

## 八、FounderOS 的终极形态

| 版本 | 形态 |
|------|------|
| V1 | 手动系统（结构化文档 + AI 辅助） |

| V3 | Agent 执行系统（AI 自动拆解任务 + 执行） |
| V4 | 自动化公司操作系统（Founder 只做关键决策） |

### 终极目标

让一个 Founder 可以管理一个越来越复杂的系统。

在 AI 和自动化时代：
- **人类提供 Vision**
- **AI 提供 Execution**
- **FounderOS 提供 Control**

最终模型：**人力 + 算力 = 文明。**
