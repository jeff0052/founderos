# FounderOS Governance

_治理结构：谁做什么决策，怎么做决策。_

---

## 一、治理哲学

FounderOS 的治理基于一个核心认知：

**创始人是善于思考但缺乏结构化执行的人。**

Jeff 的强项：
- Vision（战略思考）
- Judgment（商务判断）
- Communication（沟通能力）

Jeff 的痛点：
- 重复性跟进（像 cron 一样要一遍一遍跑的任务）
- 状态同步（各项目当前在哪个阶段）
- 时间提醒（关键节点容易遗忘）
- 信息整理（对话散落，难以形成决策 input）

**FounderOS 的治理目标：让 Jeff 专注在高价值的思考和判断上，把结构化执行交给系统。**

## 二、OKR 治理框架

### 核心设计

每个文明模块（Uniweb、Onta、License、Vibecash）采用 OKR 方式管理：

- **Objective**：Jeff 设定的清晰方向
- **Key Results**：可量化的关键成果指标
- **Missions**：AI/团队基于 KR 自动拆解的具体任务

### Jeff 的决策边界

```
┌─────────────────────────────┐
│  Objective + Key Results     │  ← Jeff 的决策层
│  （方向 + 衡量标准）          │
└──────────────┬──────────────┘
               │  Jeff 不需要下沉到这里
┌──────────────┴──────────────┐
│  Missions → Tasks            │  ← AI/团队的执行层
│  （拆解 + 执行 + 反馈）       │
└─────────────────────────────┘
```

### KR 管理原则

- **KR 数量不限** — AI 可以同时追踪 100 个 KR，不受人类注意力限制
- **信息过滤** — AI 负责过滤，只把需要 Jeff 判断的 KR 上报（通常 3-5 个）
- **异常驱动** — 正常推进的 KR 不打扰 Jeff，只有阻塞/异常才上报

```
全部 KR → AI 过滤 → 需要 Jeff 判断的 → Jeff 决策
  N个       ↓              3-5个
        自动处理 95%
```

### Jeff 每周只回答三个问题

1. **OKR 方向还对吗？**（外部信号有没有改变优先级）
2. **KR 进展如何？**（红黄绿灯）
3. **需要调整什么？**（≤3 个战略决策）

## 四、Signal → Decision → Mission 流程

### Signal 处理流程

```
外部信号进入
     ↓
AI 自动分类（市场/合作伙伴/监管/技术/投资）
     ↓
AI 评估重要性（高/中/低）
     ↓
高重要性 → 立即通知 Jeff
中重要性 → 写入 Signals 面板，等周度 Review
低重要性 → 存档，不打扰
```

### Decision 流程

```
Jeff 查看 State + Signals
     ↓
做出战略决策（≤3 个/周）
     ↓
Decision 记录为 ADR
     ↓
AI 将 Decision 拆解为 Missions
     ↓
Missions 分配 Owner + Deadline
     ↓
AI/团队执行
     ↓
Result 反馈 → State 更新
```

### 周度 Review 流程

**时间：** 每周 30-60 分钟

**顺序：**
1. **Update State** — 更新核心指标数据
2. **Review Signals** — 查看本周重要信号
3. **Make Decisions** — 做出关键决策
4. **Update Missions** — 检查任务进展

## 五、对外合作治理原则

### 原则一：合作不对称
我们提供技术 + 合规，合作伙伴提供用户 + 渠道。永远不跟合作伙伴争夺 C 端用户。

### 原则二：标准化接入
一套 API，接入所有合作伙伴。统一 SDK、安全标准、结算规则。

### 原则三：利益分享
让合作伙伴赚钱，我们赚规模。网络效应比单笔利润更重要。

### 原则四：地缘战略
每个区域找一个 anchor 合作伙伴。

### 原则五：时机选择
在合作伙伴需要我们的时候进入（监管压力时、技术瓶颈时、扩张阶段时）。

## 六、任务挂载原则

**关键 ADR：任务挂在战略地图上，不挂在组织结构上。**

❌ 错误：市场部 3 个任务、技术部 5 个任务
✅ 正确：Uniweb 7 个任务、Onta 9 个任务

**原因：**
- 组织结构思维让你关注"部门效率"
- 战略地图思维让你关注"文明进展"
- 对 Founder 来说，看清文明进展比管理部门更重要
