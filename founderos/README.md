# FounderOS

**A Control System for Founders**

契约驱动的创始人操作系统。人类经营契约，AI放大杠杆。

## 文档结构

### 📖 给人看的（Human-Readable）
| 文件 | 用途 |
|------|------|
| `vision.md` | 愿景、哲学、文明模型 |
| `strategy.md` | 战略地图、扩张路径 |
| `governance.md` | 治理结构、OKR框架、角色定义 |
| `methodology.md` | CDRE方法论（工程指南） |
| `decisions/` | ADR架构决策记录 |

### 🤖 给AI看的（Machine-Readable）
| 目录 | 用途 |
|------|------|
| `contracts/` | 数据Schema、接口定义、系统宪法 |
| `specs/` | 模块规约、行为规则、显式排除 |
| `prompts/` | AI工作指令模板 |
| `tests/` | 验证规则、验收标准 |

### 📊 运行时数据
| 目录 | 用途 |
|------|------|
| `world/` | 外部世界地图（支付网络、监管、竞争） |
| `missions/` | 按文明地图组织的任务线 |

## 核心原则

1. **代码是廉价的一次性产物，契约才是核心资产**
2. **Jeff只经营OKR层，Mission层由AI/团队拆解执行**
3. **AI是杠杆放大器，不是核心决策者**
4. **KR数量不限，但到达Jeff的信息量受控**
5. **每天打开一次，它就活着；三天不看，它就死了**
