# ADR-2026-03-13-001: FounderOS 架构决策

## Context

Jeff 是一个善于思考但缺乏结构化执行的 CEO。每天有大量连续性项目（NetStar、Benji、Paymongo、Mosaic 等）需要追踪，传统 TODO list 无法管理项目流程。需要一个系统来放大 Founder 的认知能力和执行杠杆。

## Decision

1. **FounderOS 定位为控制系统（Control System），不是软件工具**
2. **采用双层世界模型**：外部世界地图（Signal）+ 内部公司身体（Mission）
3. **任务挂在战略地图上（Uniweb/Onta/License/Vibecash），不挂在组织结构上**
4. **采用 OKR 框架**：Jeff 只管 Objective + KR，Mission 层由 AI/团队拆解
5. **KR 数量不限**：AI 做过滤器，只把需要 Jeff 判断的信息上报
6. **核心公式**：State + Signal → Decision → Action → New State
7. **文档分两类**：给人看的（叙事+逻辑）和给AI看的（形式化+精确）
8. **采用 CDRE 方法论指导技术实现**

## Consequences

✅ Jeff 可以专注高价值的思考和判断工作
✅ 结构化执行由系统保证，不依赖人类记忆
✅ 项目流程可追踪，不会遗忘
✅ AI 杠杆最大化利用
✅ 文明地图提供战略全局视角

⚠️ 系统需要每天打开才能保持活力
⚠️ 初期需要投入时间建立契约层
⚠️ OKR 设定需要 Jeff 的战略判断，AI 无法替代
⚠️ V1 版本仍为结构化文档，非产品界面
